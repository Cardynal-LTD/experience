Security

Last updated: January 2026

Our Commitment

Cardynal is built with security at its core. We protect your data and your customers' data with industry-standard practices and infrastructure.

Infrastructure

Hosting. Our infrastructure runs on Railway, using isolated containers with automatic scaling and redundancy.

Database. We use PostgreSQL with encrypted connections. Database backups are performed daily and retained for 30 days.

Network. All traffic is encrypted in transit using TLS 1.3. We enforce HTTPS on all endpoints.

Data Protection

Encryption at Rest

All stored data is encrypted using AES-256.

Encryption in Transit

All data transmission uses TLS 1.3.

Access Control

Role-based access. Employee access is logged and restricted.

Data Isolation

Customer data is logically isolated per account.

Authentication

Password Security. Passwords are hashed using bcrypt with appropriate cost factors. We never store plaintext passwords.

Session Management. Sessions use secure, HTTP-only cookies with appropriate expiration.

JWT Tokens. API authentication uses signed JWT tokens with 24-hour expiration.

Application Security

Input Validation. All user inputs are validated and sanitized to prevent injection attacks.

API Security. Rate limiting is implemented on all endpoints. Authentication is required for all non-public APIs.

Dependency Management. We regularly update dependencies and monitor for known vulnerabilities.

Operational Security

Monitoring. We monitor our systems 24/7 for anomalies and security events.

Incident Response. We have documented procedures for security incidents. Critical issues are addressed immediately.

Change Management. All code changes go through review before deployment.

Compliance

Cardynal complies with:

  • Israel's Protection of Privacy Law (Amendment 13, 2025)
  • Privacy Protection Authority guidelines on data security
  • Industry best practices for SaaS security

Employee Security

Background Checks. Key personnel undergo background verification.

Access Principles. We follow least-privilege access principles. Employees only have access to systems necessary for their role.

Training. Team members receive security awareness training.

Third-Party Security

We carefully evaluate third-party providers for security practices. Key providers include:

  • Railway — Infrastructure hosting
  • Supabase — Database services (self-hosted)
  • LLM Providers — AI processing (enterprise agreements with data protection terms)

Vulnerability Reporting

If you discover a security vulnerability, please report it to security@cardynal.io. We appreciate responsible disclosure and will respond within 48 hours.

Data Retention and Deletion

You control your data retention settings. Upon account termination, data is deleted within 30 days. You can request immediate deletion at any time.

Business Continuity

Backups. Daily automated backups with 30-day retention.

Redundancy. Multi-region infrastructure with automatic failover.

Recovery. Documented disaster recovery procedures with regular testing.

Questions?
